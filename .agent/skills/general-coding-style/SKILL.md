## コメント規約

- 実装の意図や「なぜそう実装したか」がわかりやすいコード・命名を心がけ、不要なコメントは避けてください。
- セキュリティに関する処理（時間制限、シングルショット配信など）には意図が分かるコメントを付与してください。
- 一時的な実装には必ず `TODO:` コメントを残し、理由を明記してください。
- コメントは日本語で書いてください

## 開発再開

- セッションの最初には、以下を実行し、状態を把握する
  - `git diff`, `git status` を行い、未コミットの状態などを把握する
  - github 上の issue を確認し、取り組んでいる途中の issue がないかを確認する
- 状態を把握した上で、ユーザーの指示を待ち、作業を進めること
